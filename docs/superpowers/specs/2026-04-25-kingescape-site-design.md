# KingEscape Site Design

## Goal

Build a single-page Portuguese presentation site for the KingEscape school project, explaining the game's story, protagonist, enemies, bosses, sectors, mechanics, and ending.

## Approved Direction

Use the hybrid narrative: Dante crosses three infernal bureaucracy sectors in one cohesive underworld journey. The sectors keep the project's maze, darkness, limited-vision, key-door, and survival mechanics believable while still giving the story a richer three-chapter structure.

## Visual Style

Premium narrative: cinematic dark page, strong hero, fog/lightning mood, high-contrast sections, large character cards, and polished chapter blocks. The tone should feel like a horror-adventure game pitch, not a plain school document.

## Page Structure

1. Hero: game title, logline, short pitch, navigation buttons.
2. Prologue: Dante is struck by a colossal lightning bolt due to Death's administrative error.
3. Gameplay journey: top-down maze-crawler, limited vision, flashlight/battery, keys, doors, enemies, survival.
4. Infernal sectors: Setor das Brasas, Limbo das Portas, Arquivo do Abismo.
5. Characters: Dante, A Morte, Capeta da Recepcao, Cerbero, Ceifeiros Auditores, Almas Errantes/Espectros.
6. Ending: Dante reaches the terminal, uses "Estorno de Alma", and returns alive.

## Implementation Notes

The current project is a static website with `index.html` and `style.css`. Keep it static and dependency-free. Use existing assets from `C:\Users\david\Documents\Mobile - Game\assets` where useful, and add AI concept art if available.

## Acceptance Criteria

- The page is a single polished HTML page in Portuguese.
- It explains the story and characters clearly enough for a teacher presentation.
- It works by opening `index.html` directly in the browser.
- It is responsive for desktop and mobile.
- It keeps a premium narrative visual direction with no build step.
